How it works
  1. Each product's name, category and description is encoded into a 384-dimensional vector with the all-MiniLM-L6-v2 model.
  2. Vectors are stored in Postgres using the pgvector extension, indexed with HNSW for fast cosine-distance search.
  3. Your query is embedded the same way; results are ranked by embedding <=> query_vector.
